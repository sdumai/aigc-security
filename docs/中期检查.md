# 中期检查报告

## ========== 一、课题主要任务 ==========

本课题旨在开发面向AIGC（AI Generated Content）的多模态安全检测工具，围绕"AI生成内容日益逼真、检测手段亟待系统化"这一现实需求，实现图像与视频的虚假内容检测、不安全内容识别以及多模态内容生成三大核心能力，并将其整合为一个可实际部署和运行的Web平台。课题主要任务包括：（1）系统性调研AIGC生成与检测领域的前沿方法，梳理图像/视频虚假检测与不安全内容识别的技术路线，总结现有方法的适用场景与不足。（2）设计并实现多模态安全检测平台的整体系统架构，涵盖前端交互、后端服务、模型推理三层结构，支持多种检测算法与生成模型的灵活接入。（3）复现并集成代表性的检测与生成算法，其中检测算法涵盖UniversalFakeDetect等本地推理模型与基于多模态大模型的API检测方案；生成算法涵盖Stable Diffusion（文生图）、ModelScope T2V（文生视频）等本地部署模型，以及火山引擎系列（人脸替换、属性编辑、图/视频生成）API接入。（4）完成系统部署上线、实验验证与对比分析，撰写毕业论文并完成答辩准备。（5）完成指定外文文献的翻译工作。

## ========== 二、简述开题以来所做的具体工作和取得的进展 ==========

开题以来，围绕"面向AIGC的多模态安全检测工具"这一研究方向，已在系统设计与开发、模型复现与接入、系统部署、外文翻译四个方面取得了较为完整的阶段性成果，以下按工作模块逐一说明。

一、系统架构设计与前端开发。系统采用三层分离式架构：前端层使用React 18 + TypeScript + Vite + Ant Design 5负责交互界面与数据展示；后端代理层使用Node.js + Express负责API鉴权签名、请求转发与业务逻辑编排；模型推理层使用Python + FastAPI + PyTorch承载本地部署的深度学习模型推理服务。三层通过HTTP/REST通信，前端请求后端代理，后端按需转发至本地模型服务或外部API，实现了灵活的模型接入与替换能力。前端已完成6个功能页面的开发（前端代码约5200行），包括：首页（系统概览与导航入口）、Deepfake生成页（人脸替换、人脸动画、属性编辑）、通用生成页（文生图、文生视频、图生视频）、虚假内容检测页（图像/视频的AI生成鉴别）、不安全内容检测页（图像/视频的违规内容审核）、数据管理页（历史记录管理与数据导出）。在交互设计上实现了拖拽上传、实时预览、可视化检测结果（置信度评分、安全等级标签）、响应式布局等用户体验优化，并采用CSS变量构建统一设计令牌（Design Tokens）确保全站视觉一致性。

二、后端服务开发。后端代理服务（约1056行）基于Node.js + Express实现，主要职责包括：API密钥管理与鉴权签名，封装火山引擎HMAC-SHA256签名算法，统一管理AK/SK与API Key，避免密钥暴露在前端；请求路由与转发，共定义14个POST接口，分别转发至本地模型服务或外部API；异步任务轮询，对视频生成等异步任务实现服务端轮询（间隔3秒、上限约6分钟），将异步流程封装为前端可直接使用的同步响应。具体端点包括检测类5个（UniversalFakeDetect本地检测、火山方舟图像AI鉴别、图片安全审核、视频AI鉴别、视频安全审核）和生成类8个（人脸替换、人脸动画、属性编辑、方舟文生图、本地SD文生图、本地ModelScope文生视频、方舟文生视频、方舟图生视频）。

三、本地模型复现与部署。根据指导教师"需部署本地推理模型，不能仅依赖外部API"的要求，已完成3个本地模型的复现与服务化部署（模型相关Python代码约4600行）。第一个是UniversalFakeDetect图像虚假检测模型，论文来源为Ojha et al.的"Towards Universal Fake Image Detectors that Generalize Across Generative Models"（CVPR 2023），模型架构基于CLIP ViT-L/14特征提取加线性分类头，利用预训练视觉模型的通用表征能力实现跨生成模型的泛化检测，部署为FastAPI服务运行在端口8008，支持Base64、URL、文件上传三种输入方式，输出真伪判定、置信度评分等信息。第二个是Stable Diffusion v1.5文生图模型，来源为runwayml/stable-diffusion-v1-5，基于Latent Diffusion Model，部署为FastAPI服务运行在端口8009，根据文本提示词生成图像，支持512到1024多种分辨率，GPU环境下启用model_cpu_offload节省显存。第三个是ModelScope Text-to-Video 1.7B文生视频模型，来源为达摩院的damo-vilab/text-to-video-ms-1.7b，1.7B参数量的文本到视频扩散模型，部署为FastAPI服务运行在端口8011，使用DPMSolverMultistepScheduler加速采样，支持FP16精度推理与VAE Slicing，实现了帧数据格式规范化函数将不同版本diffusers返回的数据统一为export_to_video所需格式。

四、外部API模型对接。通过火山引擎视觉智能与方舟大模型平台，在平台内对接了10项API能力，包括：人脸替换（火山视觉智能人脸融合3.6）、人脸动画（方舟Seedance 1.0 Lite I2V）、属性编辑（火山视觉SeedEdit 3.0）、文生图（方舟Seedream 4.0）、文生视频（方舟Seedance 1.0 Lite T2V）、图生视频（方舟Seedance 1.0 Lite I2V）、图像AI鉴别（方舟多模态理解Chat）、视频AI鉴别（方舟视频理解Chat）、图像安全审核和视频安全审核。鉴权方面，视觉智能接口使用AK/SK加HMAC-SHA256签名，方舟接口使用API Key Bearer Token，两套鉴权体系在后端统一封装，前端无需感知密钥细节。

五、系统部署与自动化。系统已部署至校内实验室服务器（10.102.32.144），采用前后端分离部署：前端通过npm run build构建静态资源后由http-server部署在53177端口；后端代理Node.js进程通过tmux运行在3001端口；本地模型服务分别通过tmux会话管理UniversalFakeDetect（8008）、Stable Diffusion（8009）、ModelScope T2V（8011）。编写了4套部署脚本（Shell + Expect，共约1300行），覆盖前端快速部署、后端部署、远程部署、全量部署等场景，实现构建、压缩、上传、安装依赖、启动服务的一键完成。

六、外文文献翻译。完成外文文献翻译工作，翻译论文为"Detecting AI-Generated Text, Images, and Videos: A Review of Methods and Emerging Frameworks"。该综述系统梳理了检测AI生成文本、图像和视频的多种方法与新兴框架，涵盖预训练语言模型的性能评估与统计特征提取方法、基于Transformer模型的分类方法、图像检测工具综述（内部细节检测、元数据分析、数字历史追踪、EXIF数据验证等）以及视频深度伪造检测的挑战与方法。翻译原文约34706字符，译文约11183字符，已提交至毕业论文管理系统。

七、工作量统计。前端React + TypeScript代码约5200行，后端Node.js代码约1050行，模型服务Python代码约4600行，部署脚本Shell + Expect约1300行，合计约12150行。

## ========== 三、存在的主要问题 ==========

尽管系统整体框架已搭建完成并可实际运行，但在模型复现、系统稳定性与研究深度方面仍存在以下问题。

（1）本地文生视频模型生成质量有待提升。ModelScope T2V 1.7B作为1.7B参数量的轻量级模型，在实际部署中存在生成画面出现噪声、色块等质量不稳定问题。经排查发现部分问题与diffusers库版本中export_to_video对帧数据形状的处理有关（5维数组未正确展开为逐帧列表），已针对性修复帧数据格式规范化逻辑，但模型本身的生成质量受限于参数规模，与商业API仍有明显差距。
（2）本地检测模型覆盖面有限。当前本地仅部署了UniversalFakeDetect一种图像检测模型，视频检测和不安全内容检测仍依赖方舟多模态大模型API。需进一步复现并部署更多本地检测算法（如CNNDetection、DRCT等），以增强系统的自主检测能力和研究对比深度。
（3）GPU资源与显存限制。实验室服务器GPU显存有限，同时运行多个本地模型时存在显存竞争问题。当前通过enable_model_cpu_offload和enable_vae_slicing等策略缓解，但在并发请求场景下仍可能出现显存不足错误。
（4）实验数据集尚未系统构建。原计划构建1000+多样化样本的实验数据集，目前主要通过平台生成功能临时产出样本，尚未形成结构化、可复用的标准数据集。
（5）论文撰写进度需加快。中期检查前主要精力集中在系统开发与模型部署上，论文正文各章节的撰写尚处于提纲阶段，需尽快推进。

## ========== 四、下一步的主要研究任务、具体设想与安排 ==========

下一步将围绕"补充本地模型、构建数据集、对比实验、论文撰写"四条主线展开。

（1）为本地检测模型扩充，预计2周。在已有UniversalFakeDetect基础上，再复现1至2种图像检测算法（优先选择CNNDetection或DRCT），部署为FastAPI服务并接入后端。探索本地视频检测模型的可行性（如Xception-c23），若GPU资源不足则以轻量级方案替代。同时对现有ModelScope T2V进行生成质量验证与调优，确认权重完整性后稳定输出。
（2）为实验数据集构建与对比实验，预计2至3周。利用平台生成模块（本地SD + ModelScope + API生成）批量产出虚假样本，结合公开数据集（GenImage、CIFAKE、FaceForensics++等）收集真实样本，构建1000+规模的实验数据集。设计对比实验方案，比较不同检测算法（本地vs API、单模型vs融合）在相同数据集上的准确率、召回率、F1-score等指标。记录实验数据并整理为论文可用的表格与图表。
（3）为系统优化与论文撰写，预计3至4周。完善数据管理模块（历史记录查询、结果导出等功能），优化系统性能（并发控制、错误恢复、日志完善）。按照论文大纲完成各章节撰写，包括：第一章绪论（研究背景与意义、国内外研究现状、研究内容与创新点），第二章相关技术综述（AIGC生成技术、检测方法分类、系统架构技术选型），第三章系统设计与实现（架构设计、模块设计、关键算法实现），第四章实验与分析（数据集、实验设置、结果对比、消融分析），第五章总结与展望。
（4）为答辩准备，预计1周。完成论文终稿修改与格式校对，制作答辩PPT并准备系统演示，预演答辩并准备可能的提问。
